# Portal 文案规范

这份规范适用于 `apps/portal` 的玩家页面、登录、挑战提交、提交状态、空状态和管理后台。

目标很简单：让每句话都承担清楚的功能。优先使用短标签、直接标题和事实状态；只有能帮助用户完成当前操作时，才保留说明文字。

## 基本规则

- 用短语和名词短语描述页面内容：`暂无记录`、`未开放`、`提交记录`、`上传截图`。
- 用事实描述状态，不替用户解释整条流程：`读取中…`、`提交中…`、`识别中…`、`已收到`。
- 标题说明“是什么”，说明文字只补充必要的条件、范围或限制。
- 空状态默认只显示当前状态。只有存在明确、可执行的下一步时，才增加按钮或引导。
- 错误指出具体对象和下一步：`无法读取挑战目录，请稍后重试。`
- 未来功能使用 `未开放` 等事实标签，不写发布日期、承诺或“以后会在这里显示”。
- 面向玩家少用内部流程词汇，例如审核、发放、编排、持久化和 OCR；只有用户需要理解时才出现。
- 保持中文标点和现有 Portal 的系统字体、短标题、克制语气一致。

## 避免的句式

以下句式通常只是重复页面已经表达的信息，应优先删除或改成短语：

- `这里……会……`
- `开放后会在这里显示`
- `留下每一次……`
- `正在……你的……`
- `完成后会自动……`
- `现在……已经整合，在这里……`
- `暂时无法……，请确认……，然后……`

不要为了“完整”补一段没有下一步的说明。页面已经有标题、状态或按钮时，不再用段落复述它们。

## 推荐写法

| 场景 | 推荐 | 避免 |
| --- | --- | --- |
| 空状态 | 暂无记录 | 还没有提交记录，新的进度会显示在这里 |
| 未开放 | 未开放 | 开放后会在这里显示 |
| 加载 | 读取中… | 正在读取你的玩家中心…… |
| 选择目标 | 选择挑战 | 选择一个挑战目标开始后续流程 |
| 上传 | 上传截图 | 上传一张完整截图，识别通过后进入人工核对 |
| 提交中 | 提交中… | 正在上传，请耐心等待系统处理 |
| 识别中 | 识别中… | 已提交，正在进行截图识别 |
| 读取失败 | 无法读取挑战目录，请稍后重试。 | 部分可提交内容暂时无法读取，请确认网络后重试 |
| 找不到记录 | 找不到这条提交记录。 | 暂时无法找到这条提交记录。 |

## 提交状态词汇

Portal 各页面使用以下统一显示文本。接口状态键保持英文，不在本规范中改名。

| 状态键 | Portal 文案 |
| --- | --- |
| `received` | 已收到 |
| `evidence_pending` | 保存截图中 |
| `evidence_stored` | 截图已保存 |
| `ocr_pending` | 等待识别 |
| `ready_for_review` | 等待核对 |
| `ocr_review_required` | 等待处理 |
| `approved` | 已通过 |
| `rejected` | 未通过 |
| `resubmission_required` | 需要重新提交 |

加载状态统一使用：`读取中…`、`保存中…`、`提交中…`、`识别中…`。

## Agent 修改清单

修改 Portal 文案前：

1. 先检查这份规范和 `apps/portal/pages/index.vue`、`apps/portal/pages/me.vue` 的现有语气。
2. 确认文字是在表达内容、状态、错误，还是在提供可执行的下一步。
3. 删除重复说明，优先尝试一个短标签或事实短句。
4. 如果是提交状态，复用上面的词汇，不在页面内另造说法。
5. 修改后搜索“这里”“开放后”“正在”“完成后”等高风险句式，并检查空状态、加载状态和错误状态。
